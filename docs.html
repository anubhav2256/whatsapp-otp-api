Overview REST API
Extreme Verify API allows you to programmatically send both One-Time Passwords (OTPs) and Custom Alert Messages via WhatsApp WebJS infrastructure. Every request is authenticated with your personal API key. Messages are delivered through your linked WhatsApp device.
Encryption AES-256 local sessions
Latency ~150ms average
Compliance Anti-spam enforced

Works with any programming language that supports HTTP requests. All responses are JSON formatted. Web dashboard includes live QR code linking and realtime usage analytics.

⚡ Supported Use Cases

OTP Verification Order confirmation Shipping update Delivery alert Class reminder Notes uploaded Exam alerts Payment success Withdrawal alert Fraud alert Transaction alert Account login alert
Authentication API Key
Every API request must include your unique apiKey inside the JSON payload. You can find your key on the dashboard → "Authentication Key" section. Keep it secret.
Header / Body pattern 
{
  "apiKey": "your-unique-api-key-xxxxxxxx",
  "number": "91XXXXXXXXXX",
  "otp":    "123456"
}
Important — Your WhatsApp session must be in "CONNECTED" state (green badge) before sending OTPs. Use the dashboard to scan QR code once per device.
Send OTP POST /api/send-otp
POST /api/send-otp

📦 Request Body (JSON)

Parameter Type Required Description
apiKey string ✅ yes Your personal API key from dashboard
number string ✅ yes Recipient's phone number (international format: 91XXXXXXXXXX or 9198...)
otp string / integer ✅ yes One-time password (4-6 digits recommended)

✅ Success Response (200)

{ "success": true }

❌ Error Response

{ "success": false, "error": "WhatsApp not connected" }
Send Custom Alert POST /api/send-message
POST /api/send-message

📦 Request Body (JSON)

Parameter Type Required Description
apiKey string ✅ yes Your personal API key from dashboard
number string ✅ yes Recipient's phone number (international format: 91XXXXXXXXXX)
message string ✅ yes Custom text content (e.g. "Your order has been shipped"). The branding footer is automatically appended.

✅ Success Response (200)

{ "success": true }

❌ Error Response

{ "success": false, "error": "apiKey, number, and message are required" }
Rate Limits & Policies
50
OTPs / day per user
Daily limit resets at 00:00 UTC
45s
Cooldown between OTPs
Minimum interval to avoid flooding
🧠 Intelligent throttling — If you exceed rate limits, API returns HTTP 429 with detailed message. Monthly aggregate limit is 1550 messages/month (sandbox tier). Use responsibly to prevent WhatsApp restrictions.
Error Reference
Error message Cause Resolution
Invalid API Key Key missing or incorrect Check dashboard and copy exact key
WhatsApp not connected Session not ready / expired Scan QR from dashboard or reconnect
Daily limit reached 50 OTPs already sent today Wait until next UTC day
Please wait X seconds Cooldown active Implement delay between requests
Failed to send OTP WhatsApp connection lost or number invalid Check number format (country code)
Code Examples
Integrate with your stack — copy & paste
PHP · cURL 
// PHP · cURL — send OTP via Extreme Verify API
$data = [
    "apiKey" => "YOUR_API_KEY_HERE",
    "number" => "91XXXXXXXXXX",
    "otp"    => rand(1111, 9999)
];

$ch = curl_init("https://yourdomain.com/api/send-otp");
curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);
curl_setopt($ch, CURLOPT_POSTFIELDS,     json_encode($data));
curl_setopt($ch, CURLOPT_HTTPHEADER,      ["Content-Type: application/json"]);
echo curl_exec($ch);
Node.js · Axios 
// Node.js · Axios — send OTP via Extreme Verify API
const axios = require('axios');

axios.post('https://yourdomain.com/api/send-otp', {

    number: '91XXXXXXXXXX',

.then(res  => console.log(res.data))
.catch(err => console.error(err.response?.data));
Python · requests 
# Python · Requests — send OTP via Extreme Verify API
import requests

url     = "https://yourdomain.com/api/send-otp"
payload = {
    "apiKey": "YOUR_API_KEY_HERE",
    "number": "91XXXXXXXXXX",
    "otp":    "9876"
}

response = requests.post(url, json=payload)
print(response.json())
Replace YOUR_API_KEY_HERE with the actual key from dashboard. The endpoint base URL depends on your deployment (localhost or production).

Frequently Asked Questions
Q: Is WhatsApp account at risk?
A: We enforce strict anti-spam policies. Using realistic delays and not exceeding daily quota reduces risk.
Q: How do I reconnect if session expires?
A: On your dashboard, click "RESET SESSION" button and scan fresh QR code via WhatsApp Linked Devices.
Q: Does the API support template messages?
A: You can send free-form text using the /api/send-message route. These alerts bypass strict Meta template approvals.
Q: Can I send OTP to international numbers?
A: Yes, but country code must be included (e.g., 1 for US, 44 for UK). Ensure WhatsApp number is active.
